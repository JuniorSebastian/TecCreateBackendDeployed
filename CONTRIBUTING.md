# Contributing

Gracias por contribuir a este proyecto. Estas son unas pautas rápidas para colaborar de forma ordenada:

- Abre un issue para discutir cambios grandes antes de empezar a desarrollar.
- Crea ramas descriptivas: `feature/xxx`, `fix/xxx`, `chore/xxx`, `cleanup/xxx`.
- Haz commits pequeños y con mensajes claros. Sigue el formato de commits tipo Angular (`feat:`, `fix:`, `chore:`, `docs:`).
- Antes de abrir PR, actualiza la rama `main` local y asegúrate de que los tests básicos pasen.
- Para cambios de estilo o linting, añade la configuración correspondiente (ESLint/Prettier) y documenta cómo ejecutarlos.

Si necesitas ayuda para preparar un workflow de CI o añadir ESLint, puedo preparar los archivos iniciales.
